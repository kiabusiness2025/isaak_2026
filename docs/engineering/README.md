# docs/engineering

Auditorías de progreso y bitácora de sesiones de trabajo sobre `isaak_2026`. No es
documentación de producto (`docs/product`) ni de arquitectura (`docs/architecture`) — es
el registro de **qué se hizo, cuándo, y en qué estado quedó** cada tramo de trabajo, para
que una auditoría posterior no tenga que reconstruirlo desde el historial de PRs.

- `sessions/` — un archivo por sesión/auditoría relevante, con fecha en el nombre
  (`AAAA-MM-DD-descripcion-corta.md`).

Convención heredada del repo `isaak` (legacy), que ya usaba `docs/engineering/sessions/`
con el mismo propósito.
